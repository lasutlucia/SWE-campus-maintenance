# Skill AI: UI Design

## Tujuan
Membuat halaman, navigasi, form, dan wireframe.

## Kapan Digunakan
Digunakan pada tahap pengerjaan proyek ketika melakukan aktivitas yang berhubungan dengan ui design.

## Input
- docs/requirements/specification.md

## Langkah Kerja
1. Petakan alur visual navigasi halaman untuk setiap role.
2. Rancang wireframe form pelaporan dan detail laporan.
5. Berhenti jika informasi tidak cukup.

## Output
- docs/design/ui_flow.md (Alur navigasi halaman)
- Wireframe/mockup antarmuka pengguna

## Aturan
- Jangan membuat fakta baru.
- Tandai asumsi.
- Gunakan ID requirement.
- Jangan melewati scope.

## Quality Check
- Apakah tata letak navigasi mempermudah perpindahan antar peran aktor?

## Kondisi Gagal
AI harus berhenti jika:
- Informasi input tidak lengkap.
- Terjadi konflik kebutuhan yang tidak dapat diselesaikan secara otomatis.

## Human Review
- Mahasiswa/Dosen harus memeriksa kesesuaian output terhadap spesifikasi tugas.
